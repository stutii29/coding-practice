# Coding Practice — L6/Staff IC Interview Prep

Pattern-based refresh, not random grinding. Each folder = one concept area.
Read the README/tutorial link first, then work the problems in `examples/` order
(easy → harder, same underlying pattern).

## Structure

| Folder | Pattern | Tutorial |
|---|---|---|
| [`arrays-hashing/`](./arrays-hashing) | Arrays, hashmaps, frequency counting | [NeetCode Arrays & Hashing](https://neetcode.io/practice/arrays-and-hashing) (free) |
| [`two-pointers-sliding-window/`](./two-pointers-sliding-window) | Two pointers, sliding window | [NeetCode Two Pointers](https://neetcode.io/practice/problem-list/two-pointers) · [NeetCode Sliding Window](https://neetcode.io/practice/problem-list/sliding-window) (free) |

All problems linked are on LeetCode's **free tier** — no subscription needed.

## How to use

1. Open the folder's `README.md` — 5 min concept refresher.
2. Solve each `.py` file's problem yourself on LeetCode first (don't peek).
3. Compare against the reference solution + complexity notes in the file.
4. If stuck > 15 min on a Medium, read the approach comment, then re-implement
   from scratch without looking.

More folders (trees/graphs, DP, heaps) will follow as the prep plan progresses.
